# GeoFS Addon-Pack V3.0.1 Release Notes

This update focuses on modularity, performance, and enhancing the **Failure Simulator** experience by transitioning it to a standalone, maintainable repository.

## 🚀 Key Improvements

### 1. Modular Architecture

We have successfully offloaded the internal **Failures Addon** logic to its own dedicated repository: `GeoFS-V3.9_Failure-Simulator`. This change significantly reduces the complexity of the main `main.js` file and allows for rapid, independent updates to failure mechanics without redeploying the entire pack.

### 2. Failure Simulator Enhancements

The migration brings several highly-requested features to the simulation:
- **Draggable UI**: The "FAILURES" button can now be moved anywhere on the screen. Its position is automatically saved to `localStorage` and persists between flights.
- **Screen Locks**: Improved boundary detection prevents the button from being dragged off-screen or hidden behind other UI elements.
- **Precision Gear Mapping**: Corrected the landing gear indices for a more realistic experience.
    - **Front Gear** (Index 0)
    - **Left Gear** (Index 1)
    - **Right Gear** (Index 2)
  The system now uses intelligent keyword detection to ensure the correct wheels fail on multi-bogie aircraft like the A380 or B747.
- **Safety Locking**: Probability sliders are now automatically locked while a simulation is active to prevent accidental resets during critical flight phases.

### 3. Performance & Stability

- **Code Optimization**: Removed 260+ lines of hardcoded legacy logic from the main script.
- **Dynamic Loading**: Implemented a lightweight CDN loader using Raw.Githack for more reliable asset delivery.
- **Null Safety**: Added exhaustive checks for aircraft suspension and engine data to prevent script errors on non-standard aircraft.

## 🛠️ Deployment Instructions

The Addon-Pack will automatically fetch the new failure logic on the next startup. No manual action is required for existing users.
